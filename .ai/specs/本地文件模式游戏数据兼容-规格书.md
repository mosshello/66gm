# 本地文件模式游戏数据兼容 技术规格书

## 目标
- 修复直接双击 `index.html` 时因 `file://` CORS 限制无法加载游戏数据的问题，同时保持 HTTP 环境可用。

## 影响范围
| 文件 | 原因 |
|---|---|
| `index.html` | 在应用脚本前加载可执行的数据文件 |
| `js/app.js` | 优先读取预加载数据，不存在时再回退到 JSON 请求 |
| `data/games.js` | 提供与 `games.json` 同步的 `window.GAMES_DATA` 数据 |

## 实施步骤
1. 由当前 `data/games.json` 机械生成 `data/games.js`，写入 `window.GAMES_DATA`。
2. 在 `js/app.js` 前加载数据脚本。
3. `loadGames` 优先读取 `window.GAMES_DATA`；仅在未预加载时使用 `fetch`。
4. 保留错误状态渲染，并补充 HTTP 响应状态检查。

## 验证方式
- `node --check data/games.js` 与 `node --check js/app.js`。
- 使用 `file:///F:/coding/66gm/index.html` 渲染并确认爆款、免费游戏卡片出现且控制台无 JSON CORS 错误。
- 使用本地 HTTP 服务复查页面。

## 回滚方案
- 删除 `data/games.js` 引用并恢复 `loadGames` 的原始 JSON 请求。

## 目标编辑文件清单
- `index.html`
- `js/app.js`
- `data/games.js`
