# 首页首屏浏览导向与游戏条修复 技术规格书

## 目标
- 将首页首屏从直接购买导向改为先浏览、先体验，并确保滚动游戏封面条在桌面端和移动端稳定可见。

## 影响范围
| 文件 | 原因 |
|---|---|
| `index.html` | 调整首屏文案、按钮层级和游戏条位置 |
| `css/style.css` | 压缩首屏高度、增加双按钮与解锁提示样式、恢复移动端游戏条 |
| `js/app.js` | 将滚动游戏条渲染与游戏列表数据请求解耦 |

## 实施步骤
1. 首屏标题突出“1000+ 免广告小游戏”，主按钮指向游戏列表，次按钮指向免费专区。
2. 将价格购买信息降级为按钮下方的轻提示，不再作为首屏视觉中心。
3. 将滚动游戏封面条前移并缩小卡片尺寸，确保常见桌面和移动视口可见。
4. 页面初始化时独立渲染游戏条；游戏列表加载成功后不重复渲染。
5. 尊重减少动态效果的系统设置，暂停滚动动画。

## 验证方式
- 检查 HTML、CSS 与 JavaScript 语法及目标选择器。
- 在本地 HTTP 服务中验证桌面端和移动端首屏：标题、双按钮、解锁提示及滚动封面条均可见。
- 验证游戏数据请求失败不影响滚动封面条生成。

## 回滚方案
- 回退上述三个目标文件中本规格对应的首屏结构、样式和初始化调用。

## 目标编辑文件清单
- `index.html`
- `css/style.css`
- `js/app.js`
